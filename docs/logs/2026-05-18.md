# Development Log - 2026-05-18

## Changes

- **[locale]**: Fixed dayjs locale not applying on mount when `locale` is set (e.g. demo `calendarKey` remount). `lastLocaleProp` is initialized as `undefined` so the locale effect runs once and calls `dayjs.locale()` and `currentDate.locale()`.
- **[i18n]**: Removed unused weekday translation keys (`sunday`–`saturday`, then `sun`–`sat`) from `Translations` and `defaultTranslations`. Calendar headers use dayjs `format('ddd'|'dddd')` with the active locale.
- **[recurrence]**: Weekly “Repeat on” labels use `weekDay.format('dd')` with `currentLocale` instead of `t('sun')`…`t('sat')`.

## Files Modified

- `src/hooks/use-calendar-engine.ts` - `lastLocaleProp` initialized to `undefined`
- `src/hooks/use-calendar-engine.test.ts` - Locale on mount and on prop change
- `src/lib/translations/types.ts` - Dropped unused weekday keys
- `src/lib/translations/default.ts` - Dropped unused weekday defaults
- `src/features/recurrence/components/recurrence-editor/recurrence-editor.tsx` - Dayjs `format('dd')` for repeat-on labels
- `src/features/recurrence/components/recurrence-editor/recurrence-editor.test.tsx` - Updated weekday label assertions

## Notes

- Demo `calendarKey` includes `locale`, which remounts `IlamyCalendar` on every locale change. Without applying locale on mount, `dayjs.locale()` never ran because `lastLocaleProp` matched the prop immediately.
- Week/month/year grid headers on this branch use dayjs formatting, not translation keys (remote branch has pending `t('sun')` wiring for those views).
