- **[feat/scroll-time]**: Address issue #132. Add `scrollTime?: string` to `IlamyCalendarProps` (inherited by `IlamyResourceCalendarProps`). New `useScrollToTime` hook scrolls to the row or cell matching the configured time on all hour-resolution views: day, week, resource day vertical, resource week vertical (block: 'start'), resource day horizontal, resource week horizontal (inline: 'start'). Hour-floor only for v1, clamps silently to the nearest visible row when the requested hour is outside the visible range (e.g. with `hideNonBusinessHours`). API name matches FullCalendar's `scrollTime` (verified via WebFetch of fullcalendar.io docs).

## Notes

- Scroll anchor uses a `data-hour="HH"` attribute on the time gutter cells (vertical) and time-header cells (horizontal), not the `data-testid`. Decouples production scroll behavior from test infrastructure and lets one selector (`[data-hour]`) drive both axes from a single hook.
- Scrolls the Radix ScrollArea viewport directly with `Element.scrollTo({ top })` / `scrollTo({ left })` rather than `scrollIntoView`. `scrollIntoView` walks up the ancestor chain and would also scroll the parent page; targeting the viewport scopes the scroll to exactly one container.
- Scroll offset is computed relative to the FIRST hour row (`targetRect.left - firstRect.left`), not the viewport's edge. The first row naturally sits past any sticky-left column (Resources) or sticky-top header, so this lands the target hour at the visible start of the time area rather than behind the sticky column.
- Requires a fixed calendar height. With `height: auto`, the internal time grid grows to fit all hours and there is no scroll container to scroll, so the prop silently no-ops (verified against FullCalendar `height` / `contentHeight` docs).
- `IlamyCalendar` and `IlamyResourceCalendar` use `...props` spread to their providers, so no passthrough changes were needed in the component shells.
