# Development Log - 2026-06-27

## Changes

- **[calendar/header]**: Code-review follow-ups on the header redesign (PR #225):
  - `calendar.tsx`: renamed `cellClass`/`dayButtonClass` to `getCellClass`/
    `getDayButtonClass` (verb prefix for value helpers); lifted the two-clause
    `cn` condition to a named `hoverBand`; replaced the multi-line
    `highlightedWeek` ternary with an `if`-assignment.
  - Extracted the duplicated month/year picker cell into a shared
    `PickerGridCell` (`header/picker-grid-cell.tsx`); `month-grid` and
    `year-grid` now consume it instead of repeating the button + `cn` classes.
  - Dropped the redundant `size="default"` on the `ViewSwitcher` toggle (it is
    the default variant) and corrected the stale "overflow menu" comment in
    `use-export-calendar.ts`.
  - `header-date-picker.tsx`: replaced the `title`/`content` IIFE `if`-cascades
    with view-keyed plain value maps (`Partial<Record<string, string>>` /
    `Partial<Record<string, ReactNode>>` + `?? <day form>` fallback).
  - Extracted `PickerNav` (`components/ui/picker-nav.tsx`): the prev/center-label/
    next header that was duplicated in `calendar.tsx`, `month-grid.tsx`, and
    `year-grid.tsx`. All three now consume it.
- **[calendar/header + context]**: Agenda view's header title + picker now mirror
  the matching grid view based on the agenda **window** (read off the view's
  `navigationStep`): `day` → day title + day picker, `week` → week range title +
  week picker (band + hover, exactly like week view), `month` → `MMM YYYY` title
  + `MonthGrid`. A custom N-day window has no grid equivalent, so it shows the
  listed range as the title and uses the day picker. Exposed the engine's
  computed visible range as `currentRange` (typed `DateRange`) on the context API
  (`IlamyCalendarApi.currentRange`, additive) to source the custom-window range.
