# Development Log - 2026-05-18

## Changes

- **[locale]**: Fixed dayjs locale not applying on mount when `locale` is set (e.g. demo `calendarKey` remount). `lastLocaleProp` is initialized as `undefined` so the locale effect runs once and calls `dayjs.locale()` and `currentDate.locale()`.
- **[i18n]**: Removed unused weekday keys (`sunday`–`saturday`, `sun`–`sat`) and month keys (`january`–`december`) from `Translations` / `defaultTranslations`. Date labels use dayjs `format()` instead of `t()` where the global locale applies.
- **[recurrence]**: Weekly “Repeat on” labels use `weekDay.format('ddd')` (replaces `DAY_KEYS` + `t('sun')`…`t('sat')`).
- **[date-utils]**: Added `isDayBeforeMonth`, `formatWeekdayDayMonth`, `formatShortMonthDay`, `formatWeekPickerRange` (compact when same month; both ends with month when the week crosses a month), and `getWeekdayHeaderLetter` (first letter of `format('dd')`, uppercased).
- **[header / title-content]**: Month/day/week pickers use dayjs helpers with `currentLocale` only where needed (`isDayBeforeMonth` / explicit `day.locale()` for `Intl`-driven ordering).
- **[year-view]**: Mini-calendar weekday headers use shared `getWeekdayHeaderLetter` + `getWeekDays` (replaces hardcoded initials and the local inline helper).
- **[ui/calendar]**: Date-picker popover grid drops `WEEKDAY_LABELS`; uses `getWeekdayHeaderLetter` + `getWeekDays`, relying on global `dayjs.locale()` from `useCalendarEngine` (no `locale` prop on `Calendar`).
- **[ui/date-picker]**: Reads `firstDayOfWeek` via `useSmartCalendarContext` selector (scalar return, not object destructure); passes it to `Calendar`; trigger label uses `format('ll')` instead of hardcoded `MMM D, YYYY`.
- **[day-view]**: Header date uses `format('LL')` via dayjs `localizedFormat` plugin.
- **[types]**: `CalendarContextType.currentLocale` is required (`string`), matching `useCalendarEngine` (fixes CI `string | undefined` errors).

## Notes

- Demo `calendarKey` includes `locale`, which remounts `IlamyCalendar` on every locale change. Without applying locale on mount, `dayjs.locale()` never ran because `lastLocaleProp` matched the prop immediately.
- **Breaking for consumers**: custom `translations` objects no longer require weekday or month name keys; use the `locale` prop + dayjs locales for month/day names.
- `getWeekdayHeaderLetter` and grid headers depend on `dayjs.locale()` set by the engine; `getWeekDays(dayjs(), …)` creates instances after that global locale is active.
- `title-content` helpers still take `currentLocale` for `Intl` (`isDayBeforeMonth`) and explicit `day.locale()` where ordering differs from a plain `format()`.
- Week/month grid column headers still use `day.format('ddd')` (dayjs locale).
- `useSmartCalendarContext(selector)` returns the selector’s return type (`T`), not the full context — e.g. `(ctx) => ctx.firstDayOfWeek` yields a `number`, so assign it directly rather than destructuring.
