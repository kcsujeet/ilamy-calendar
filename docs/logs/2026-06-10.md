# Development Log - 2026-06-10

## Changes

- **[week-view]**: Fixed a stale-render bug — the `columns` `useMemo` deps array listed `visibleDays.map` (a stable function reference) instead of `visibleDays`, so day columns never recomputed when `hiddenDays` changed after the initial render. Test-first (red-green).
- **[year-view]**: Fixed mini-calendar week-start mismatch — day cells were generated with `startOf('week')` (locale week start) while the weekday headers respected `firstDayOfWeek`, misaligning cells under headers whenever the prop differed from the locale default. Now uses `getWeekDays(monthStart, firstDayOfWeek)` like the month view. Test-first (red-green).
- **[all-events-dialog]**: Fixed a `$$` typo in the `DraggableEvent` `elementId` template (`all-events-dialog-event-$123` → `...-123`), removed a dead `adjustedFirstDayOfCalendar` computation block and a misleading comment. Added the dialog's first behavioral test (opens via "+N more", lists all events of the day) in `month-view.test.tsx` — the identifier itself is not DOM-observable, so the typo fix rode under that new coverage as a refactor.
- **[code style]**: Mechanical Clean Code / project-rule sweep across packages and demo:
  - Removed all `export default` (header components `Header`/`TitleContent`/`ViewControls`, demo `App`, demo `seed.ts` `dummyEvents`) in favor of named exports; updated importers.
  - Flattened all nested ternaries: `draggable-event.tsx` cursor classes (named `cn()` variables), `use-autocomplete-timepicker.ts` 12-hour conversion (if/else), `position-day-events.ts` cluster offsets (lookup map replacing magic ternary chain), `demo-calendar-settings.tsx` initial-date select value (extracted `resolveInitialDateOption` resolver).
  - Converted bracket array access to `.at()` with proper guards (no non-null assertions): `use-calendar-engine.ts`, `date-utils.ts` (`getMonthWeeks`), `position-day-events.ts`, `title-content.tsx`, `resource-event-grid.tsx`, `use-scroll-to-time.ts`, `recurrence-presets.ts`, `recurrence-editor.test.tsx`, demo `demo-data.ts`.
  - `event-form.tsx`: introduced `DEFAULT_EVENT_COLOR` constant instead of `COLOR_OPTIONS[0].value`.
  - `base-header.tsx`: export filename now built with `dayjs().format('YYYY-MM-DD')` instead of `new Date().toISOString().split('T')[0]`.
  - Replaced `!!` with `Boolean()` (`view-hours.ts`, `useProcessedWeekEvents.ts`); replaced the `#root` non-null assertion in demo `frontend.tsx` with an explicit guard.
