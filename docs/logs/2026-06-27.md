# Development Log - 2026-06-27

## Changes

- **[calendar/header]**: `ViewSwitcher` active pill uses `bg-primary` /
  `text-primary-foreground` so it stands out in light and dark mode; the track
  gets a `p-0.5` inset.
- **[calendar/grid]**: Border fixes (surfaced by the theme's translucent border).
  - Vertical day separators moved from the column wrapper to the cells
    (`border-r`): a translucent border on the opaque flex column was overpainted
    by the neighbor at fractional pixel widths and rendered faint.
  - The all-day row no longer passes `isLastRow` (it isn't the last row — the
    time grid follows), so its cells keep `border-b`; dropped the redundant
    wrapper border.
  - Edge cells no longer double the calendar body's outer `border`: month's last
    column cell drops `border-r`; the time grid's last hour row drops `border-b`
    (both the gutter and day cells). Combined with the existing last-row /
    last-column strips, all four calendar edges are single-bordered. Specs added
    for each edge.
- **[demo]**: Adopted the current shadcn (Tailwind v4) token palette in
  `apps/demo/src/styles/globals.css`. The dark `--border`/`--input` are
  translucent white, so grid lines stay visible over filled (hover/disabled)
  cells. The old palette set `--border` == `--secondary` == `--accent`, hiding
  borders under cell fills. Library code unchanged: any consumer with a proper
  theme just works.

- **[calendar/header]**: Code-review follow-ups on the header redesign (PR #225):
  - `calendar.tsx`: renamed `cellClass`/`dayButtonClass` to `getCellClass`/
    `getDayButtonClass` (verb prefix for value helpers); lifted the two-clause
    `cn` condition to a named `hoverBand`; replaced the multi-line
    `highlightedWeek` ternary with an `if`-assignment.
  - Extracted the duplicated month/year picker cell into a shared
    `PickerGridCell` (`header/picker-grid-cell.tsx`); `month-grid` and
    `year-grid` now consume it instead of repeating the button + `cn` classes.
  - Dropped the redundant `size="default"` on the `ViewSwitcher` toggle (it is
    the default variant) and corrected the stale "overflow menu" comment in
    `use-export-calendar.ts`.
  - `header-date-picker.tsx`: replaced the `title`/`content` IIFE `if`-cascades
    with view-keyed plain value maps (`Partial<Record<string, string>>` /
    `Partial<Record<string, ReactNode>>` + `?? <day form>` fallback).
  - Extracted `PickerNav` (`components/ui/picker-nav.tsx`): the prev/center-label/
    next header that was duplicated in `calendar.tsx`, `month-grid.tsx`, and
    `year-grid.tsx`. All three now consume it.
- **[calendar/header + context]**: Agenda view's header title + picker now mirror
  the matching grid view based on the agenda **window** (read off the view's
  `navigationStep`): `day` → day title + day picker, `week` → week range title +
  week picker (band + hover, exactly like week view), `month` → `MMM YYYY` title
  + `MonthGrid`. A custom N-day window has no grid equivalent, so it shows the
  listed range as the title and uses the day picker. Exposed the engine's
  computed visible range as `currentRange` (typed `DateRange`) on the context API
  (`IlamyCalendarApi.currentRange`, additive) to source the custom-window range.

- **[playground]**: Surfaced the agenda window as a contextual segmented
  switcher. The Agenda Window control (Day / Week / Month / 3-day / 14-day) is
  now a `FormButtonGroup` shown when Agenda is the selected Initial View, instead
  of an always-visible `Select`. Added `parse` support to `FormButtonGroup` so
  the numeric windows store as numbers (matching `FormSelect`). The agenda plugin
  was already registered live via `createDemoPlugins(agendaWindow)`.
- **[ui]**: Added shadcn `ButtonGroup` (`@ilamy/ui/components/button-group.tsx`:
  `ButtonGroup` / `ButtonGroupText` / `buttonGroupVariants`, vendored from the
  shadcn registry; `ButtonGroupSeparator` omitted as the repo has no `Separator`).
  `FormButtonGroup` is rebuilt on it (a connected row of `Button`s, selected =
  filled, others = outline).
