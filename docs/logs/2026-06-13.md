# Development Log - 2026-06-13

## Changes

- **[recurrence/types]**: Added a required `deleted: CalendarEvent[]` field to `PluginMutationResult` (rows to persist via `onEventDelete`) and widened `applyDelete` to `CalendarEvent[] | PluginMutationResult`, mirroring `applyEdit`.
- **[recurrence handler]**: `updateRecurringEvent` now returns `deleted` on every branch; the "all" edit reports the detached overrides it drops (Google-Calendar reset) so consumers can delete those stale rows. `deleteRecurringEvent` now returns a structured `PluginMutationResult` instead of a bare array, reporting real stored-row deltas per scope.
- **[calendar data hook]**: `applyScopedEdit` now also dispatches `deleted -> onEventDelete`. `applyScopedDelete` was rewritten to mirror the edit path: it dispatches `updated/added/deleted` from a structured result, falling back to the old single-`onEventDelete` path for bare arrays.

## Files Modified

- `packages/types/src/index.ts` - Added `deleted` to `PluginMutationResult`; widened `applyDelete` return type.
- `packages/recurrence/src/utils/recurrence-handler.ts` - `deleted` on all `updateRecurringEvent` returns; `deleteRecurringEvent` returns `PluginMutationResult` with per-scope deltas.
- `packages/calendar/src/features/calendar/hooks/use-calendar-data.ts` - `deleted` loop in `applyScopedEdit`; structured-result handling in `applyScopedDelete`; updated deps.
- `packages/recurrence/src/utils/recurrence-handler.test.ts` - delete wrapper + new tests for `deleted`/`updated` per scope.
- `packages/recurrence/src/utils/crud-operations.test.ts` - delete wrapper for array-shape assertions.
- `packages/calendar/src/features/calendar/hooks/use-calendar-engine.test.ts` - new tests: scope-all delete cascades `onEventDelete`; scope-this delete fires `onEventUpdate` for the base EXDATE.
- `docs/migration-v2.md` - documented the `deleted` field, `applyDelete` widening, and the scoped-delete behavior change.

## Notes

- `deleted` contents per scope (delete): this -> dropped override (or `[]`); following -> `[]`; all -> base + every override. `updated`: this -> base with new EXDATE; following -> terminated base; all -> `[]`.
- edit "all" now reports dropped detached overrides in `deleted` (was silently filtered out of `events`).
- Engine tests require a build first (recurrence package resolves `@ilamy/calendar` via built `dist`) and must run from `packages/calendar` for the happy-dom preload.
- The recurrence plugin's `applyDelete` needed no logic change — the structured result flows through unchanged once the type widened.
- `PluginMutationResult` is unreleased (introduced earlier in this same PR/branch), so making `deleted` required is non-breaking.
