# Development Log - 2026-05-18

## Changes

- **[locale]**: Fixed dayjs locale not applying on mount when `locale` is set (e.g. demo `calendarKey` remount). `lastLocaleProp` is initialized as `undefined` so the locale effect runs once and calls `dayjs.locale()` and `currentDate.locale()`.
- **[i18n]**: Removed unused weekday keys (`sunday`–`saturday`, `sun`–`sat`) and month keys (`january`–`december`) from `Translations` / `defaultTranslations`. Date labels now use dayjs `format()` with `currentLocale` instead of `t()`.
- **[recurrence]**: Weekly “Repeat on” labels use `weekDay.format('dd')` with `currentLocale` (replaces `DAY_KEYS` + `t('sun')`…`t('sat')`).
- **[year-view]**: Mini-calendar weekday headers use `weekDay.format('dd').charAt(0).toLocaleUpperCase()` via `getWeekDays` (replaces hardcoded `DAY_HEADER_NAMES`).
- **[header]**: Month picker in `title-content` uses `currentDate.month(index).format('MMMM')` with `currentLocale ?? 'en'` (replaces `MONTH_KEYS` + `t('january')`…`t('december')`).

## Files Modified

- `src/hooks/use-calendar-engine.ts` - `lastLocaleProp` initialized to `undefined`
- `src/hooks/use-calendar-engine.test.ts` - Locale on mount and on prop change
- `src/lib/translations/types.ts` - Dropped unused weekday and month keys
- `src/lib/translations/default.ts` - Dropped unused weekday and month defaults

- `src/features/calendar/components/year-view/year-view.tsx` - Locale-aware weekday header letters
- `src/components/header/title-content.tsx` - Month picker uses dayjs `MMMM` with locale

## Notes

- Demo `calendarKey` includes `locale`, which remounts `IlamyCalendar` on every locale change. Without applying locale on mount, `dayjs.locale()` never ran because `lastLocaleProp` matched the prop immediately.
- **Breaking for consumers**: custom `translations` objects no longer require weekday or month name keys; use the `locale` prop + dayjs locales for month/day names.
- Week/month grid column headers still use `day.format('ddd')` (dayjs locale). Only recurrence “repeat on”, year-view initials, and header month picker were migrated in this session.
