# Development Log - 2026-05-18

## Changes

- **[locale]**: Fixed dayjs locale not applying when the calendar mounts with a non-default `locale` prop (e.g. demo remounts via `key` that includes locale). `lastLocaleProp` now starts as `undefined` so the locale effect runs on initial mount; event `start`/`end` are re-localized alongside `currentDate`.

## Files Modified

- `src/hooks/use-calendar-engine.ts` - Apply locale on mount; propagate locale to event dates
- `src/hooks/use-calendar-engine.test.ts` - Locale mount, change, and event tests

## Notes

- Demo `calendarKey` includes `locale`, which remounts `IlamyCalendar` on every locale change. Without applying locale on mount, `dayjs.locale()` never ran because `lastLocaleRef` matched the prop immediately.
