# Development Log - 2026-04-26

## Changes

- **triage**: Reviewed PR #98 (`feature/hide-hours-in-week-day-view` by mark-tutton) to plan takeover.
- **merge**: Merged `main` into `feature/hide-hours-in-week-day-view` (commit `272e20e`). Three conflicts resolved.
- **refactor**: Removed the consumer-passed `expandAllDayRow` prop. Decision is now auto-detected inside `VerticalGrid` itself.
  - First attempt (CSS-only flex priority): rejected after visual review — broke the "small but non-zero hours" case (e.g. 9am–10am).
  - Second attempt (CSS-only via `:has()` and DOM markers): worked but verbose (6 inline arbitrary variants). Rejected as too clever for the value.
  - Final: `VerticalGrid` derives `expandAllDayRow = columns.every(c => !c.days?.length)` and forwards to `VerticalGridHeaderContainer`. Consumers pass nothing; the prop is internal-only.
- **resource calendar fix**: Extended the empty-hours expansion to cover the resource variants (the actual review feedback on PR #98). Initially tried keeping the ScrollArea wrapping the header for resource (to preserve horizontal scroll of resource columns), but visual verification showed all-day didn't expand: Radix's Viewport wraps children in a `<div style={{minWidth: '100%', display: 'table'}}>`, and the existing `*:min-h-full` viewport class only sets a min-height. `h-full` on the header inside resolved against a parent without a definite height and fell back to `auto` (= 60px content size), so all-day's `flex-1` had nothing to grow into. Final: the early-return path is now shared by both variants. Trade-off: in the (rare) empty-hours case with many resources that don't fit horizontally, the resource header loses horizontal scroll. Acceptable since there's no body to align with anyway. `DayView` (regular) and all three resource-vertical views (`resource-day-vertical`, `resource-week-vertical`, `resource-month-vertical`) now inherit the fix automatically.
- **cleanup**: Removed the now-dead `{!expandAllDayRow && (…)}` guard around the body div in `VerticalGrid`'s normal render path. With the early-return firing for both variants when `expandAllDayRow` is true, the second path is only reached when `expandAllDayRow` is false — so the body always renders there. Misleading comment about resource scrolling was also dropped.
- **comment**: Added a 3-line comment above the `expandAllDayRow` derivation in `VerticalGrid` so a reader understands the trigger condition (`hideNonBusinessHours` collapsing the visible hour range to zero) without having to grep `useProcessedDayEvents` / `view-hours.ts`.
