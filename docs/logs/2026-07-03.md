# Development Log - 2026-07-03

## Changes

- **[calendar/props]**: Added `onMoreEventsClick?: (day: Dayjs, events: CalendarEvent[]) => void` prop. When provided, clicking the "+N more" overflow indicator in a grid cell invokes the callback (with the cell's day and full event list) instead of opening the built-in all-events dialog. When omitted, the default dialog still opens. Applies to both regular and resource calendars (unified under one `CalendarProvider` in v2).
- **[grid-cell]**: `showAllEvents` now early-returns to the consumer callback when set; otherwise falls back to the dialog ref.
- **[docs]**: Documented the new prop in `types-and-interfaces.md`.
- **[playground]**: Added `handleMoreEventsClick` and threaded `onMoreEventsClick` through `SharedCalendarProps` into both calendar variants.
