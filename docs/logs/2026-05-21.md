# Development Log - 2026-05-21

## Changes

- **[week-view / layout]**: Replaced calc-based day column widths (`w-[calc((100%-4rem)/var(--visible-days))]`) with `flex-1` so week columns share space evenly inside the grid.
- **[week-view / overflow]**: Added `min-w-0` on week header cells, horizontal grid rows, and `GridCell` so flex children can shrink instead of forcing horizontal overflow.
- **[vertical-grid / borders]**: Moved day-column `border-r` from per-hour wrappers to `VerticalGridCol` (last column `border-r-0`); hour stacks no longer duplicate side borders.
- **[grid-cell]**: Simplified cell border classes (`border-r border-b min-w-0`, removed `last:border-r-0 only:border-r` combo).
- **[week-view / time column]**: Time gutter uses `border-r-0` on the sticky left column to avoid double borders against the first day column.
- **[demo]**: Default demo view set to `week`; seed events disabled (`dummyEvents = []`) for empty-grid layout testing.
- **[styles]**: Added `--spacing: 0.5rem` to `:root` in `globals.css`.

## Notes

Flex grids need `min-w-0` on columns for truncation/shrink to work; without it, `min-width: auto` on flex items preserves content width and breaks week view on narrow viewports. Staged commit scope: grid + week-view only (`grid-cell`, `horizontal-grid-row`, `vertical-grid-col`, `week-view`). Demo and `globals.css` changes are local QA helpers and may stay unstaged.
