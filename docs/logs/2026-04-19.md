## Demo label a11y cleanup

15 `lint/a11y/noLabelWithoutControl` warnings in `demo-calendar-settings.tsx` were blocking the IDE. Each label sat above a group of custom React components (`<Button>`, `<SelectTrigger>`). Biome's rule only detects raw HTML form controls as "inside a label", so even restructuring the JSX to nest controls under the label didn't silence it.

- First attempted: convert `<label>` to `<div>` (rejected — labels ARE semantically correct here).
- Then: proper restructure to `<label>` wrapping its control block (valid HTML, but Biome still warns because `<Button>` isn't a raw `<button>`).
- Final: Biome override scoped to `src/components/demo/**` disables just `noLabelWithoutControl` there. Library code still enforces the rule.

Post-fix bundle: 600.05 KB raw / 163.40 KB gzip (no meaningful change from the restructure).

## Bundle architecture discovery + entry-point split (in progress)

Further bundle analysis surfaced a key finding: **bunup already externalizes every dep** (rrule, motion/react, clsx, dayjs, radix-ui, etc. are all `import` statements in the output, not inlined). The 600 KB bundle is almost entirely our own source code (~10,324 lines of TS/TSX).

Empirical measurements via temporary src/index.ts edits:
- Full exports: 600.05 KB raw / 163.40 KB gzip
- Only `IlamyCalendar`: 520.68 KB / 146.92 KB
- Only `IlamyResourceCalendar`: 535.48 KB / 148.24 KB
- Consumers who import only one calendar variant carry ~76 KB of the other variant they'll never use.

**Motion cost:** only 2.7 KB raw when stubbed — tree-shakes extremely well. Not a worthwhile swap target.

### Entry-point split — considered and rejected

Prototyped splitting `IlamyResourceCalendar` into `@ilamy/calendar/resource` subpath (~76 KB savings for consumers using only the regular calendar). Rejected as too disruptive — would require a 2.0 major-version release, breaking every current consumer of `IlamyResourceCalendar`. Not worth the churn for 76 KB.

Reverted: removed `src/resource.ts`, restored resource exports in `src/index.ts`, reverted `bunup.config.ts` to single entry. Bundle back at 600.01 KB raw / 163.38 KB gzip.

Remaining bundle-reduction levers all look small (<10 KB) and/or architectural. Pausing this line of work.
