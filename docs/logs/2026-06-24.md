# Development Log - 2026-06-24

## Changes

- **[calendar/header]**: Replaced the month/year/week/day title dropdowns in the
  header with `Select` instead of a `Popover` + list of `Button`s. Each picker is
  now a controlled `Select` whose options carry the `Dayjs` they navigate to;
  `onValueChange` looks the chosen option up and calls `selectDate`. The trigger
  keeps each view's distinct title format (e.g. the day picker's "Monday, 5") by
  rendering custom trigger content instead of `SelectValue`, with `aria-label`
  set to the title so the combobox has an accessible name.

- **[calendar/header]**: Fixed the day picker's "Today" badge layout. With the
  fixed `w-40` content the date ("Jun 24, 2026") wrapped and the badge got jammed
  in. The `SelectContent` now sizes to its widest item (`min-w-40`), items are
  `whitespace-nowrap`, and the badge is a proper pill (`px-1.5 py-0.5 leading-none`).

- **[demo/website]**: Added a CSS rule to both consumer global stylesheets so
  Radix popups (Select/Dialog/Popover) no longer hide the page scrollbar / shift
  layout on open. react-remove-scroll locks body scroll with
  `body[data-scroll-locked] { overflow: hidden !important; margin-right: <gap>px !important }`;
  the doubled `[data-scroll-locked][data-scroll-locked]` selector (specificity
  0,2,0 > the lib's 0,1,1) forces `overflow-y: scroll` + `margin-right: 0`. This
  affects every Radix Select in the app (recurrence editor, event form, settings,
  header), not just the header. See shadcn-ui/ui#4227.

- **[calendar/header]**: Stopped the header `Select` trigger from keeping a focus
  ring after closing. Radix returns focus to the trigger on close (via
  `FocusScope`), which left a `:focus-visible` ring until the next click;
  `onCloseAutoFocus={(e) => e.preventDefault()}` on the header `SelectContent`
  suppresses the focus-return. Scoped to the header (not the shared `SelectContent`).

## Notes

- The header `Select` triggers intentionally have no associated `<label>`, so the
  combobox accessible name comes from `aria-label` (set to the current title). The
  recurrence/event-form selects get their name from a `<Label htmlFor>` instead.
- Custom trigger content (not `SelectValue`) is what preserves the per-view title
  format; Radix Select doesn't require `Select.Value` for the trigger to function.
