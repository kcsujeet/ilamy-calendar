# Development Log - 2026-05-12

## Changes

- **horizontal-grid**: Added the "now" line (current time indicator) to horizontal resource hour-grids. Parameterized the existing `CurrentTimeIndicator` with an `axis?` prop (`'vertical' | 'horizontal'`, default `'vertical'`) so the same component handles both layouts. Mounted the indicator inside `HorizontalGridEventsLayer`, gated on `gridType === 'hour'` — naturally excludes regular MonthView and resource MonthView (both day-resolution) while including resource day horizontal and resource week horizontal hourly views.
- **public-api**: Added required `axis: 'vertical' | 'horizontal'` field to `RenderCurrentTimeIndicatorProps`. Lets a single consumer-supplied render function position itself correctly across vertical and horizontal calendars by branching on `axis` inside the callback. Minor breaking change for TS consumers that construct the props as literals (rare — mostly test scaffolding).

## Files Modified

- `src/features/calendar/types/index.ts` — added required `axis` field to `RenderCurrentTimeIndicatorProps`.
- `src/components/current-time-indicator.tsx` — added `axis?` prop with default `'vertical'`, switched the default-render branch to swap `top`/`left` and `h-0.5`/`w-0.5` based on axis, forwarded `axis` to the custom render callback.
- `src/components/horizontal-grid/horizontal-grid-events-layer.tsx` — accepted `gridType` and `resource` props, mounted `CurrentTimeIndicator` with `axis="horizontal"` gated on `gridType === 'hour'`. Computed `rangeStart` / `rangeEnd` from `days` mirroring the vertical layer.
- `src/components/horizontal-grid/horizontal-grid-row.tsx` — threaded `resource` and `gridType` through both the flat-row events layer and the `GroupedColumn` variant (used by resource week horizontal hourly).
- `src/components/current-time-indicator.test.tsx` — added 4 test cases covering horizontal default rendering (left/w-0.5), hidden-when-out-of-range, axis forwarded to custom render, axis defaults to `'vertical'`.
- `src/components/demo/demo-page.tsx` — updated the playground's custom `renderCurrentTimeIndicator` to branch on `axis`, demonstrating consumer-side axis handling. Horizontal branch swaps the styling to `w-0.5 top-0 bottom-0` and positions via `left: ${progress}%`, with the time badge anchored to the top of the column instead of the left edge of the row.

## Notes

Addresses upstream issue #121.

Gating chosen as `gridType === 'hour'` rather than `variant === 'resource'` because the former also correctly excludes resource MonthView (day-resolution) where a 24h-percentage line per cell would be a meaningless 1px sliver. Single-condition gate, no extra `variant` plumbing needed into the events layer.

In the resource-week-horizontal (grouped columns) path, the indicator is mounted in every column's events layer. The component's own `isWithinRange` check naturally suppresses rendering in non-today columns — no separate "is this today's column?" logic needed.
