# Development Log - 2026-06-26

## Changes

- **[calendar/header]**: Full header redesign. Navigation (grouped prev/next,
  Today, and the title-as-dropdown) on the left; view + actions on the right.
  Responsive via **CSS container queries + `flex-wrap`** (no JS measurement):
  - `flex-wrap` keeps everything on one row whenever it fits and spills to a
    second row only when too narrow. The inner row is `justify-center` by
    default and `@lg/base-header:justify-between` once the compact row fits
    (~512px), so phones wrap to two centered rows while everything from tablet
    up spreads to the edges. The title sits in a single shared cluster so it is
    never duplicated.
  - View control swaps at `@3xl/base-header` (768px): segmented `ViewSwitcher`
    above, compact `ViewMenu` dropdown below. Export/New show their labels at
    `@2xl` and collapse to icon-only below. Both control variants live in the
    DOM (one `display:none`); in happy-dom the segmented radios and the Export
    button are the unique matches (menus stay closed), so tests are unambiguous.
- **[calendar/header]**: The title is a single context-aware date picker
  (`HeaderDatePicker`, an outline `Button` trigger so its border/height match
  the other header buttons): day/week open a day grid, month opens a Jan–Dec
  `MonthGrid` (year pager), year opens a `YearGrid` (12-year block pager). Week
  titles spanning two years drop the year (see use-date-time-formatters).
- **[calendar/ui]**: Extended the shared `Calendar` for the week picker, all
  **pure CSS** (no JS hover state):
  - `weekHover` highlights the hovered week as one continuous bar via a per-row
    `group/week` + `group-hover/week:bg-accent` (rounded only at the row ends).
  - `highlightedWeekOf` statically marks the current week with a solid
    `bg-primary` bar (mirrors the day picker's selected day) and stays put while
    a different week is hovered.
  - "today" switched from a filled cell to an outline ring; width uses the
    `w-72` token (was an arbitrary `w-[280px]`); day-cell rendering extracted to
    `DayCell` + `dayButtonClass`/`cellClass`.
- **[calendar/header]**: `ViewMenu` uses each view's own `PluginView.icon`
  (Day `Square`, Week `Columns3`, Month `Grid3x3`, Year `Grid2x2`, Agenda
  `List`) on both the trigger and the menu items, instead of one shared icon.
  `useHeaderViews` filters out views without `supportsResources` when the
  calendar has resources (shared by switcher + menu).
- **[calendar/header]**: `ViewSwitcher` height comes from the toggle's
  `default` size variant (token-only, no hand-picked `h-7`/`h-full`); the
  control totals `h-9` to match the action buttons.
- **[calendar/hooks]**: `formatDateRange` drops the year for ranges that span
  two years. `Intl.formatRange` otherwise injects the year on BOTH ends
  ("Dec 30, 2026 – Jan 5, 2027") which wraps the header; a cross-year week
  always spans two different months, so "Dec 30 – Jan 5" stays unambiguous.
  Same-year ranges keep `formatRange`'s concise collapsing.
- **[ui]**: Added shadcn `DropdownMenu` (`@radix-ui/react-dropdown-menu`) and
  `ToggleGroup` (`@radix-ui/react-toggle-group`, toggle variants inlined) to
  `@ilamy/ui`. Both also declared in `@ilamy/calendar` deps for externalization
  and ignored in `fallow.toml` (reached only via bundled `@ilamy/ui`).
- **[i18n]**: Added `previous` / `next` keys to the default translations (nav
  button aria-labels).

## Notes

- **Container queries + `flex-wrap`, no JS**: an early pass used a ResizeObserver
  tier hook and JS hover state; both were replaced with the repo's existing
  container-query / pure-CSS approach. The single shared title avoids the
  duplicate-DOM problem that motivated the JS attempt.
- **No arbitrary Tailwind values**: picker widths use `w-72`; the segmented
  control's height comes from the size variant rather than a raw `h-*`. shadcn's
  own tabs use `h-9 p-[3px]` + `h-[calc(100%-1px)]` for an inset track; we drop
  the inset to stay token-only (active segment fills the full height).
- **Calendar (shadcn) not adopted** for the pickers: shadcn's Calendar pulls
  `react-day-picker` + `date-fns`, clashing with this repo's dayjs-only
  convention. Reused the existing dayjs `Calendar` instead.
- Verified: 816 calendar tests pass, type-check + biome clean, full ordered
  build clean (new radix deps externalize as bare imports, no broken chunks).
- Needs browser verification (not happy-dom-testable): responsive tier switching,
  the context-aware picker popovers, and the week-band hover/active styling.
