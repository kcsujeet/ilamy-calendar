## Changes (PR #194 review follow-up)

- **[recurrence handler — edit "this"]**: Editing an already-detached stored override no longer fires a redundant no-op `onEventUpdate` on the base. The branch now computes `baseChanged = nextExdates !== existingExdates || baseEvent.uid !== seriesUid` and reports the base in `updated` only when it actually changed. In a consistent store (base already exdates the occurrence and carries the series uid) `updated` is just `[detachedOverride]`; first-time/inconsistent stores still report `[base, override]`.
- **[recurrence handler — edit "following"]**: Detached overrides of the series whose occurrence falls after the split were left orphaned in the store. Added an `isFollowingOverride` predicate (mirrors `isDetachedOverrideOfSeries`); the terminated base now drops exdates beyond `until`; the orphaned overrides are excluded from `events` and reported in `deleted`. Return is `{ events, updated: [terminatedEvent], added: [newSeriesEvent], deleted: followingOverrides }`.
- **[issue #156 regression pins]**: Added engine-level characterization tests asserting scoped recurring DELETE fires callbacks carrying the BASE id (never a generated `_0`): this → `onEventUpdate` base + exdate, no delete; following → `onEventUpdate` base + `until`, no delete; all → `onEventDelete` base. These pin already-correct behavior on HEAD and passed as-is (no bug revealed).

## Changes (PR #194 self-review refactors)

- **[recurrence handler — delete dispatch]**: Converted `deleteRecurringEvent`'s `switch (scope)` to a `scopedDeleteHandlers` lookup map (`deleteThisScope`/`deleteFollowingScope`/`deleteAllScope`), mirroring `updateRecurringEvent`'s `scopedUpdateHandlers`. Both functions now dispatch the same scope union the same way and share the single `Invalid scope` throw shape. Behavior-preserving; pinned by the existing per-scope `events`/`updated`/`added`/`deleted` assertions.
- **[recurrence handler — replaceStoredOverride]**: Changed from 5 positional args (three adjacent same-typed `CalendarEvent`) to a single `ReplaceStoredOverrideArgs` object param, matching the `ScopedUpdateContext` pattern. Removes the call-site transposition risk the types could not catch.
