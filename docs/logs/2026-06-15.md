# Development Log - 2026-06-15

## Changes

- **[issue #205]**: Investigated "Resource view month doesn't highlight current date". The bug is in the reporter's published `1.8.1` (released 2026-06-06); it is already fixed on `main`. The today-highlight in `ResourceMonthHorizontalHeader` was added by commit `c1d562b` (2026-06-13, "share CurrentTimeIndicator + DayLabel via @ilamy/ui"), which post-dates the `v1.8.1` tag — at `v1.8.1` the month view had zero `isToday`/`DayLabel` usage. No production fix needed; the fix ships in the next release.
- **[calendar]**: Added a regression test guarding the resource month (horizontal) today highlight — there was no test for it, which is why it regressed/was-missing. The resource week header was already covered; the month header was not.
- **[ui]**: `DayLabel` now exposes `data-today="true"` on its wrapper when `today` is set, as a stable styling/selection hook (and a non-brittle test target — avoids asserting `bg-primary` classes in JSDOM). `DayLabel` is the single source of truth for today-highlighting, so this hook applies across all views/plugins.

## Files Modified

- `packages/ui/src/components/day-label.tsx` - Added `data-today` attribute on the wrapper + doc note on the `today` prop.
- `packages/calendar/src/features/calendar/components/views/month.test.tsx` - New `ResourceMonthHorizontal` regression test: renders the horizontal resource month and asserts exactly one `[data-today="true"]` cell, showing today's day number.

## Notes

- Root-cause method: `git merge-base --is-ancestor c1d562b v1.8.1` returns false (fix not in 1.8.1); `git show v1.8.1` of the month view shows no today-highlight code.
- In the horizontal resource month arrangement only the header renders a today-aware `DayLabel` (body grid cells use `showDayNumber=false`), so exactly one element carries the highlight — the test relies on that invariant.
