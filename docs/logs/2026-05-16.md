# Development Log - 2026-05-16

## Changes (pending commit)

- **scope `"this"` — override id & uid**: detached override id is `<baseId>_modified_<ts>` (not `<instanceId>_modified_<ts>`). Override uses `baseEvent.uid` when set; otherwise parent and override both get `${baseEvent.id}@ilamy.calendar` (parent uid is not rewritten when already present).
- **`findBaseEventIndex`**: `getSeriesParentIdFromEventId` parses parent id from instance (`_N`) or override (`_modified_`) ids when uid-based lookup fails (e.g. instance without uid).
- **`generateRecurringEvents`**: skip EXDATE / detached-override slots using calendar-day matching (not exact ISO), so scope `"all"` time shifts do not re-materialize a series instance on an excepted day.
- **scope `"all"`**: apply series field updates to detached overrides; include them in `updated` for `onEventUpdate`.
- **`updateRecurringEvent` callbacks**: `onEventUpdate` receives stored rows (`updated`: base + EXDATE, terminated series, or scope-`all` base); `onEventAdd` receives new rows (`added`: detached override or split series). No longer fires `onEventUpdate` with synthetic instance `_0`.
- **`RecurringUpdateResult`**: `{ events, updated, added }` from `updateRecurringEvent`; exported from package index.
- **CI / tests**: Biome format; `String(event.id)` for id parsing; typed `onEventUpdate`/`onEventAdd` mocks in `provider.test.tsx`; duplicate-override test expects 1 Jan-3 event (EXDATE + override only); scope-`all` time test expects `rrule.dtstart` aligned with new start.
- **seed**: explicit `uid` on daily check-in recurring demo event.
