# Development Log - 2026-06-23

## Changes

- **[event-form]**: Migrated the event form to **react-hook-form + valibot**
  (`@hookform/resolvers/valibot`). Replaced ~10 `useState` hooks + manual
  `onChange`/validation with a single `useForm` + valibot schema. Validation
  errors now render (previously a failed check silently blocked submit).
- **[ui]**: Vendored a focused subset of shadcn's `Field` primitive
  (`Field`, `FieldLabel`, `FieldError`) into `@ilamy/ui` for accessible
  label/error markup.
- **[calendar/components/form]**: New form-agnostic, context-bound field
  components (`FormInput`, `FormSelect`, `FormCheckbox`, `FormDatePicker`,
  `FormTimePicker`, `FormColorPicker`) plus a shared `FieldWrapper` that owns the
  label/error scaffold. Each reads `control` from `FormProvider` via
  `useController` — no `control` prop and no form-type generic at call sites.
- **[event-form]**: Decomposed the form into logical sections
  (`EventDateTimeFields`, `EventResourceField`, `EventColorField`,
  `EventFormFooter`). Sections pull form state through `useFormContext` /
  `useWatch` instead of props; `event-form.tsx` is now a thin composition root
  (499 → ~150 lines).
- **[event-form-utils]**: Collapsed `buildDateTime` + `buildEndDateTime` into one
  pure `buildDateTime(date, time)`; the all-day boundary (`00:00`/`23:59`) moved
  to the submit mapper.
- **[event-form-schema]**: Added `toEventFields` (form values → `CalendarEvent`
  fields) and extracted `persistEvent` so `onSubmit` stays simple.

## Files Modified

- `packages/calendar/package.json`, `bun.lock` - added react-hook-form@7.80.0,
  valibot@1.4.1, @hookform/resolvers@5.4.0 (declared deps; bundled-external, so
  not in dist).
- `packages/ui/src/components/field.tsx` - new (vendored Field subset).
- `packages/calendar/src/components/form/*` - new field components + FieldWrapper.
- `packages/calendar/src/features/calendar/components/event-form/*` - schema,
  sections, rewritten `event-form.tsx`.
- `packages/calendar/src/features/calendar/utils/event-form-utils.ts` + test -
  deduped date/time builder.

## Notes

- **Bundle impact**: published `@ilamy/calendar` dist grew **+1.3 KB gz** (own
  code + vendored Field). The three form libs are external `dependencies` (bunup
  externalizes everything in dependencies/peerDependencies by default), so they
  are NOT in dist — but they enter the consumer's bundle (~13 KB gz). Demo build:
  274.96 → 289.04 KB gz.
- This is an evaluation/refactor spike; the bundle-vs-benefit tradeoff is the open
  question for review (does the +~14 KB gz consumer cost justify the cleaner form
  + reusable field kit + rendered validation errors).
- Behavior preserved: the existing 619-line event-form integration spec stays
  green (808 calendar tests). The 3 removed util unit tests (all-day branch) were
  relocated into the all-day submission integration test.
