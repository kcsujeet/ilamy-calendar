# Development Log - 2026-05-13

## Changes

- **public-api**: Relaxed `axis` on `RenderCurrentTimeIndicatorProps` from required to optional, per upstream maintainer review feedback on PR #122. The library always populates the field, so consumers see a defined value; the optional type just stops breaking strict-mode literal-constructor consumers. Honors the "No public-API changes" commitment from issue #121.
- **tests**: Added two integration tests in `horizontal-grid-row.test.tsx` locking in the `gridType === 'hour'` gate — one verifies the indicator mounts when `gridType` is `'hour'`, the other verifies it does not when `gridType` is `'day'`.

## Files Modified

- `src/features/calendar/types/index.ts` — flipped `axis` field to optional; tightened docstring to note the library always populates it.
- `src/components/horizontal-grid/horizontal-grid-row.test.tsx` — added a `current time indicator gating` describe block with the two integration tests.

## Notes

Addresses both blocking comments from kcsujeet on PR #122.

The events layer doesn't accept a `now` prop (the underlying `CurrentTimeIndicator` falls back to real-time `dayjs()` when `now` isn't passed), so the hour-grid test constructs its columns against `dayjs().startOf('hour')` plus the next hour — a ~2-hour window keeps the test resilient to second-level timing drift across CI/HMR runs.
