# Development Log - 2026-06-22

## Changes

- **[drag-to-create]**: Orientation-aware edge auto-scroll. The gesture hook now
  takes a `scrollAxis` (`'x' | 'y' | 'both'`) and zeroes the off-axis scroll
  delta, so a horizontal resource grid can't auto-scroll vertically across
  resources (and vice versa). The provider derives it: `'both'` for regular
  calendars, and the resource layout axis (`vertical -> 'y'`, else `'x'`) when
  resources are present.
- **[drag-to-create]**: Selection mirror now renders *inside* the grid's scroll
  content instead of as a body-portal overlay. It mounts into the new
  `data-calendar-scroll-content` box (positioned `absolute`, `z-15`), so it
  scrolls with the cells, is clipped by the scroll viewport, and is painted
  beneath the sticky header (`z-21`) and resource column (`z-20`). This fixes the
  overlay covering the resource-calendar header/columns. Falls back to the
  previous body-fixed overlay (clipped to `data-calendar-viewport`) when no
  scroll content is found.
- **[calendar]**: Exposed the resolved `orientation` (`'horizontal' | 'vertical'`)
  on the public `IlamyCalendarApi` so plugins can read the layout axis through
  the public context rather than reaching into internals.
- **[calendar]**: Added `data-calendar-scroll-content` (and `relative`) to the
  vertical/horizontal grid scroll-content boxes as a stable plugin mount contract
  (mirrors the existing `data-calendar-viewport` precedent).

## Notes

- The auto-scroll, mirror placement, and axis-lock depend on real layout
  (`getBoundingClientRect`, scroll offsets, stacking) that happy-dom can't model,
  so they need visual verification on the demo. The pure helpers (`computeEdgeScroll`,
  `unionRect`, `intersectRect`) are unit-tested; `orientation` on the public API is.
- Stacking relies on the mirror (`z-15`), cells (`z-10`), resource column
  (`z-20`), and header (`z-21`) sharing a stacking context. The mirror mounts into
  the same scroll-content box as the column, so the column directly outranks it;
  the header is a sibling of that box at `z-21` and outranks the whole subtree.
