# Development Log - 2026-06-07

## Changes

- **[monorepo/phase-3]**: Extracted two shared leaf packages to finish the plugin
  boundary and kill duplication.
  - `@ilamy/utils` — the configured `dayjs` instance (`./dayjs`) plus shared
    helpers `safeDate`/`omitKeys`/`listKey` (`./helpers`). Created because the
    configured dayjs's plugin type augmentations (`.utc()`, `.tz()`,
    `.isSameOrBefore()`) are **lost across the dist `.d.ts` boundary**: bunup
    emits the export as plain `typeof dayjs`, so any consumer of
    `@ilamy/calendar`'s `dayjs` export (incl. third-party plugins) lost those
    methods. Plugins now import dayjs from `@ilamy/utils/dayjs` (source) and get
    the augmentations. Also dedup'd recurrence's local helper copies.
  - `@ilamy/types` — the plugin-contract types (`CalendarEvent`, `WeekDays`,
    `BusinessHours`, `IlamyPlugin`, `PluginMutationArgs`, `PluginDateRange`,
    `PluginView`, `EventFormSlotContext`, `EventMutationScopeSlotContext`,
    `Dayjs`). Canonical definitions live here; `@ilamy/calendar` re-exports them
    from the old `@/` locations (thin shims) so core call sites are unchanged.
- **[monorepo/recurrence]**: `@ilamy/calendar-recurrence` now resolves
  `@ilamy/calendar` via its built dist `.d.ts` (faithful dogfooding — the
  164-error "@/ flood" from pointing at source is gone). Contract types are
  imported from `@ilamy/calendar` (NOT `@ilamy/types`) because the
  `CalendarEvent` `declare module` augmentation must target the package whose
  runtime exposes events; bunup inlines `CalendarEvent` into calendar's dist, so
  the runtime (`useIlamyCalendarContext().rawEvents`) and the augment must agree
  on that identity.
- **[recurrence/ui]**: Rewrote the recurrence editor's frequency `Select` to the
  Radix compound API (`SelectTrigger`/`SelectValue`/`SelectContent`/`SelectItem`)
  and the edit dialog to the Radix `Dialog` API (`onOpenChange`, no `onClose` on
  `DialogContent`).
- **[recurrence/tests]**: Fixed `bun:test` type resolution (added
  `bun-env.d.ts` with `/// <reference types="bun" />`; dropped the package-local
  `@types/bun` so it uses the hoisted root copy). Updated `selectFrequency` to
  drive the Radix Select (open trigger, click option). Converted the 7
  controlled-`rerender` editor tests to unmount+remount: happy-dom + Radix +
  React 19 loops on `rerender()` of a controlled Radix Select (nested composed
  refs re-attach — "Maximum update depth"); a fresh mount asserts the same
  `value -> display` output. Not a real-browser bug; not fixed by happy-dom
  18/19/20, Radix patch bumps, or DOM shims.
- **[calendar/tests]**: Removed the 4 recurrence-integration tests from
  `event-form.test.tsx` (and the `recurrencePlugin` import). They created a
  source/dist dual-`CalendarContext` split (calendar source provider vs. the
  plugin's dist-context hook). Plugin integration is covered in the recurrence
  package against the public API; calendar core stays plugin-agnostic.
- **[build/lint]**: calendar bunup externalizes all `@ilamy/*` siblings.
  biome `noRestrictedImports` (the dayjs ban) now exempts
  `packages/utils/src/dayjs.ts` and `packages/types/src/index.ts`.
