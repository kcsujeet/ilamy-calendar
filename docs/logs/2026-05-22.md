# Development Log - 2026-05-22

## Changes

- **[i18n / date display]**: Removed custom locale format helpers from `date-utils` (`isDayBeforeMonth`, `formatWeekdayDayMonth`, `formatShortMonthDay`, `formatWeekPickerRange`). Header week/day pickers now use dayjs `format('ll')` so ordering and month names follow the active locale without duplicate `Intl` logic.
- **[header / title-content]**: Week title, day picker rows, and week-range list labels use `day.format('ll')` / `start.format('ll') - end.format('ll')`; dropped `currentLocale` prop threading where no longer needed.
- **[tests]**: Removed obsolete `date-utils.test.ts` suites for the deleted helpers; dropped unused `dayjs/locale/fr.js` test import.
- **[cleanup]**: Removed unused `useMemo` imports in `event-form.tsx` and `year-view.tsx`; unused `WeekDays` import in `ilamy-calendar.tsx`.

## Notes

`dayjs` locale must be set on the calendar engine (`currentLocale` / `dayjs.locale()`) for `format('ll')` to localize header dates. Prefer one formatting path (dayjs) over parallel `Intl` + custom order helpers. `date-utils.test.ts`: 58 tests pass after helper removal.
