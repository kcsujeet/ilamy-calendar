This is a temporary repro aid for issue #172 ("Blank Resource View after navigating to an empty month") so calendar navigation persists into the demo's `initialDate` state and switching to Resource View mounts with the same date. - **[demo / calendarKey]**: Removed `initialDate.toISOString()` from `calendarKey`. With `handleDateChange` now setting `initialDate` on every navigation, including the timestamp in the key forced a full calendar remount on every prev/next click, which snapped the view back to `initialView` ('month') even when the user was on Week or Day. Key now only includes `locale`, `initialView`, `timeFormat`, `useCustomTimeIndicator`. Navigation no longer remounts; the calendar's own state handles the date. The `initialDate` is still seeded across calendar-type switches because the JSX renders different components (`IlamyCalendar` vs `IlamyResourceCalendar`), which triggers a fresh mount independent of the key.

With `border-r-0` on the column AND `border-r` on the per-cell wrapper, the right border now comes from a single source (the cells), matching the week view's convention.
