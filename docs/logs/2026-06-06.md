# Development Log - 2026-06-06

## Changes

- **[recurrence] #151 / PR #176**: Scope-"all" recurring edit now RESETS the
  series (Google-Calendar semantics). Previously, editing a series with scope
  "all" left behind any pre-existing detached override of the same series,
  duplicating the occurrence alongside the reset base. The `case 'all':` branch
  now clears `exdates` on the base and removes same-series detached overrides
  via a named `isDetachedOverrideOfSeries` predicate (review request: no inline
  multi-clause negation). The submitted wall-clock time is anchored to the
  base event's original date (duration preserved when no end is submitted); the
  `applyAllScopeUpdates` doc comment was corrected to describe the wall-clock
  rule (it already used `withTimeOfDay`, the comment had stale "delta" wording).

## Files Modified

  `applyAllScopeUpdates` doc comment; `case 'all':` now writes the anchored base
  with `exdates: undefined` and returns the list filtered through
  `isDetachedOverrideOfSeries` to drop same-series detached overrides.
- `src/features/recurrence/utils/crud-operations.test.ts` - Added compact

  base date with duration preserved, and a regression test documenting the
  intentional resurrection of a previously-deleted occurrence (delete one with
  scope "this" → EXDATE, then edit "all" → `exdates` undefined).

## Notes

- The resurrection of previously-deleted/overridden occurrences on an "all"
  edit is intentional, matching Google Calendar: a whole-series edit collapses
  the series back to a single base row with no exceptions.
