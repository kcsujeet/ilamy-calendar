# Development Log - 2026-04-04

## Changes

- **[perf]**: Removed Framer Motion (`motion` package) entirely from the library.
  - `AnimatedSection` (AnimatePresence + motion.div) was the last remaining Framer Motion usage, wrapping every `DraggableEvent`. With ~30 events, this created 120 Framer Motion components, each triggering Presence lifecycle effects that cascaded into 31 commits on every date navigation.
  - Replaced with CSS animations via `tailwindcss-animate` (`animate-in fade-in zoom-in-95`) — same visual effect, zero React overhead.
  - Deleted `src/components/animations/animated-section.tsx` and removed `motion` from `package.json`.
  - Bundle: 592KB (down from 597KB). One fewer dependency for consumers.

- **[perf]**: Wrapped date/view navigation in `React.startTransition`.
  - `navigatePeriod`, `selectDate`, `today`, and `handleViewChange` in `useCalendarEngine` now use `startTransition` to mark navigation as non-urgent. React batches the cascading `useDraggable` registration effects more aggressively, keeping the UI responsive during transitions.

- **[refactor]**: Extracted shared utilities and components to reduce bundle size and duplication.
  - `isEventInRange()` — extracted from 4 inline overlap check blocks in `use-calendar-engine.ts` and `useProcessedWeekEvents.ts`.
  - `useStableResources()` + `getResourceBusinessHours()` — extracted from 2 + 5 inline blocks across resource view files.
  - `useCalendarHandlers()` + `useAllEventsDialog()` — extracted identical callback handlers from both `CalendarProvider` and `ResourceCalendarProvider`.
  - `createTimeColumn()` — factory for the firstCol time column object, extracted from 4 view files.
  - `ResourceVerticalHeader`, `ResourceAllDaySection`, `TimeHeaderRow` — shared components extracted from identical JSX in resource views.
  - Shared class constants (`RESOURCE_CORNER`, `RESOURCE_ROW_LABEL`, `TIME_COLUMN`, `TIME_COLUMN_CELL`, `CELL_CLASS`, `TODAY_HIGHLIGHT`, `HEADER_ANIMATION`) — extracted from repeated className strings across the codebase.

- **[refactor]**: Simplified `GridCell` and `DroppableCell`.
  - Merged `GridCellWithEvents` back into `GridCell` — the split was no longer needed since events come pre-computed via props.
  - `DroppableCell` now reads `onCellClick`, `disableDragAndDrop`, `disableCellClick`, `classesOverride`, `view` directly from context instead of receiving them as props.

## Notes

- Framer Motion was the root cause of 31 commits on date navigation — each DraggableEvent's AnimatePresence triggered Presence lifecycle effects that cascaded through DndContext.
- CSS animations via tailwindcss-animate provide the same visual effect (fade-in + scale) with zero React overhead since they run on the browser's compositor thread.
- The shared class constants use flat `const` exports (not an object) because the bundler can inline and tree-shake them more efficiently.
- `startTransition` doesn't reduce commit count in the profiler but keeps the UI responsive — React can interrupt the transition if another navigation happens.
- Bundle progression: 612KB → 601KB → 597KB → 592KB across all optimization rounds.
